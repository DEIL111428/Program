# Лабораторная работа № 5

## Формулировка задания
Необходимо сравнить время выполнения двух реализаций функции вычисления факториала:

- рекурсивной,
- итеративной (через цикл).

Также требуется изучить принцип **мемоизации** и реализовать сравнение **мемоизованных** и **немемоизованных** версий обеих реализаций.

Для измерения времени использовать модуль timeit, для визуализации — matplotlib. Построить график зависимости времени выполнения от входного значения n.

## Условия выполнения

- Реализованы функции:
    - fact_recursive(n) — рекурсивный вариант без мемоизации,
    - fact_iterative(n) — итеративный вариант без мемоизации,
    - fact_recursive_memo(n) — рекурсивный вариант с мемоизацией (@lru_cache),
    - fact_iterative_memo(n) — итеративный вариант с ручной мемоизацией через словарь.
    
- Используется **фиксированный набор входных значений**: n = 5, 30, 55, ..., 500 (шаг 25).
- Для каждого значения n проводится несколько прогонов (repeat=5), и выбирается **минимальное** время (для уменьшения влияния шумов системы).
- Визуализация: график с четырьмя линиями — по одной для каждого варианта реализации.

## Описание основных функций

#### fact_recursive(n)
Рекурсивно вычисляет факториал без кэширования. Базовый случай: n == 0 или n == 1.

#### fact_iterative(n)
Вычисляет факториал в цикле от 2 до n.

#### fact_recursive_memo(n)
Рекурсивная версия с декоратором @lru_cache(maxsize=None) для автоматической мемоизации.

#### fact_iterative_memo(n, cache={})
Итеративная версия с ручным кэшированием промежуточных результатов в словаре cache.

#### benchmark(func, n)
Используется внутри цикла для измерения времени выполнения одного вызова функции.

## Результаты и выводы

[![image.png](https://i.postimg.cc/zBxBbGfr/image.png)](https://postimg.cc/TLWGBGm7)
На графике видно, что:

- **Итеративная реализация** стабильно быстрее рекурсивной при больших n из-за отсутствия накладных расходов на вызовы функций.
- **Мемоизация** в рекурсивной версии значительно ускоряет повторные вызовы, но в рамках одного вызова (fact_recursive_memo(n)) эффект минимальный, так как все промежуточные значения вычисляются впервые.
- **Итеративная мемоизация** даёт выигрыш только при многократных вызовах с разными n, но в текущем бенчмарке (один вызов на `n`) преимущество незначительно.
- При n > 400 рекурсивная реализация может начать страдать от ограничения глубины рекурсии (в Python по умолчанию ~1000), хотя в данном диапазоне это не критично.

**Вывод**: для однократного вычисления факториала предпочтительна **итеративная реализация.** Мемоизация оправдана только при многократных вызовах с пересекающимися аргументами.
