# Лабораторная работа № 6
## Формулировка задания
Написать функцию _get_currencies(currency_codes, url)_, которая обращается к API по _url_ (по умолчанию - [https://www.cbr-xml-daily.ru/daily_json.js](https://www.cbr-xml-daily.ru/daily_json.js)) и возвращает словарь курсов валют для валют из списка _currency_codes_.

В возвращаемом словаре ключи - символьные коды валют, а значения - их курсы.
В случае ошибки запроса функция должна вернуть None.
Для обращения к API использовать функцию get модуля requests.

##### Итерация 1
Предусмотреть в функции логирование ошибок с использованием стандартного потока вывода (sys.stdout).

Функция должна обрабатывать следующие [исключения](https://docs.python.org/3/tutorial/errors.html):
- в ответе не содержатся курсы валют;  
- в словаре курсов валют нет валюты из списка currency_codes;  
- ошибка выполнения запроса к API.  

##### Итерация 2.
Вынести логирование ошибок из функции _get_currencies(currency_codes, url)_ в [декоратор](https://docs.python.org/3/glossary.html#term-decorator).
##### Итерация 3.
Оформить логирование ошибок с использованием модуля logging.
Документация по logging: [https://docs.python.org/3/library/logging.html](https://docs.python.org/3/library/logging.html)
Туториал по logging: [https://docs.python.org/3/howto/logging.html](https://docs.python.org/3/howto/logging.html)
##### Тестирование функций должно содержать:
- проверку ключей и значений возвращаемого словаря;  
- проверку обработки исключений;  
- проверку записей логов в поток вывода.
## Ход выполнения
### Итерация 1 

#### Как реализовано:
- Всё в одной функции.
- Используется try...except **внутри самой функции**.
- Все сообщения об ошибках выводятся через print(..., file=sys.stdout).

#### Цель:
Показать **базовую обработку ошибок**:
- HTTP-ошибки (сетевые, 404 и т.п.)
- Отсутствие поля "Valute" в ответе
- Отсутствие запрошенной валюты

#### Минусы:
- Логика обработки ошибок "размазана" по коду.
- Нет разделения ответственности: функция и делает работу, и сама себя логирует.
- Нельзя легко переиспользовать логику логирования в других функциях.

### Итерация 2 

#### Как реализовано:
- Создан **декоратор @log_errors**, который оборачивает функцию.
- Внутри функции **нет try...except** — всё перехватывает декоратор.
- Всё ещё используется print(..., file=sys.stdout) (не logging).

#### Цель:
Показать **DRY-принцип** и **разделение ответственности**:
Функция делает своё дело, а логирование - задача декоратора.

#### Минусы:
- Всё ещё не использует стандартный механизм логирования Python (logging).
- Нет уровней логирования (warning, error и т.д.).

### Итерация 3 
#### Как реализовано:
- Используется модуль logging.
- Настроено через basicConfig() с форматом времени, уровня и сообщения.
- Логгер создаётся как logger = logging.getLogger(__name__) 
- Декоратор теперь логирует через logger.error(), logger.warning() и т.д.

#### Цель:
Показать **профессиональный подход к логированию**:
- Можно легко перенаправить логи в файл, сеть, системный журнал.
- Есть уровни важности: INFO, WARNING, ERROR.
- Поддержка структурированного вывода и фильтрации.
- Совместимость с другими модулями — все могут использовать logging.
